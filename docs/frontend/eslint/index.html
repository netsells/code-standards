# Eslint

We make use of an automated approach of enforcing JavaScript code standards. These standards are constantly evolving to ensure we are making use of the best methods, and in order to catch more edge cases ensuring our code is consistent across the board.

# Rules

The following rules are currently configured.

Rule Description Link
arrow-spacing Requires spacing before and after arrow functions arrow Link
comma-dangle Requires trailing commas when the last element or property is in a different line than the closing ] or } and disallows trailing commas when the last element or property is on the same line as the closing ] or } Link
curly Forces formatting of curly brace conventions Link
dot-location Discourage placing the dot on the property rather than the property Link
dot-notation Forces using dot notation exclusively for getting object properties. Link
eol-last Force new lines at end of files Link
eqeqeq Forces equality operators to be type-safe Link
key-spacing Enforces spacing around the colon in object literal properties Link
no-alert Disallow alert() function Link
no-cond-assign Discourage conditional assignment of variables Link
no-console Disallow using the console Link
no-debugger Disallows debugger statements Link
no-duplicate-imports Disallow duplicate imports Link
no-else-return Disallow else blocks after return statements in if statements Link
no-empty-function Disallow empty functions Link
no-empty Disallow empty block statements Link
no-eval Disallow the use of the eval() function Link
no-floating-decimal Disallow floating decimals. Link
no-implicit-coercion Discourage using confusing and sometimes unreadable JS tricks to do simple functions. Link
no-lonely-if Discourage if statements as the only statement in else blocks Link
no-mixed-spaces-and-tabs Disallow mixed spaces and tabs for indentation Link
no-multiple-empty-lines Restricts the number of allowed empty lines Link
no-restricted-imports Disallows importing specific libraries. We disallow lodash as a whole in favour of specific imports Link
no-sparse-arrays Disallows array literals with empty slots Link
no-template-curly-in-string Throw a warning when a regular string contains a text which looks like a template literal placeholder Link
no-unneeded-ternary Disallows ternary operators when simpler alternatives exist Link
no-unused-vars Prevents leaving unused imports & vars in code Link
no-use-before-define Prevents use of an identifier that has not yet been declared Link
no-useless-return Reports the use of redundant return statements Link
no-var Discourage using 'var' for creating variables - require using let/const instead Link
prefer-arrow-callback Forces use of ES6 arrow function expressions Link
prefer-const Flags variables that are defined using 'let' but then never reassigned Link
prefer-template Encourage using template literals instead of '+' operator on strings Link
quotes Enforce the use of single quotes when using JavaScript Link
semi Require a semi-colon at the end of every line Link
space-before-function-paren Require a space before function parenthesis Link
template-curly-spacing Throw a warning when a regular string contains a text which looks like a template literal placeholder Link
valid-jsdoc Require jsdoc data to be consistently valid Link
yoda Discourage code typed like yoda would speak Link
require-jsdoc-except/require-jsdoc Exclude certain methods from requiring JSDoc definitions. Link
vue/attribute-hyphenation Force attributes to be hyphenated rather than camelCase Link
vue/html-indent Enforce 4 space continuous indentation Link
vue/html-quotes Force double quote style in html syntax Link
vue/max-attributes-per-line Limits the maximum number of attributes/properties per line to improve readability. Link
vue/mustache-interpolation-spacing Expect one space between expression and curly brackets. Link
vue/name-property-casing Force kebab-case component names Link
vue/no-dupe-keys Disallow duplicate key names to avoid overwriting Link
vue/no-duplicate-attributes When duplicate arguments exist, only the last one is valid Link
vue/no-multi-spaces Remove multiple spaces in a row between attributes which are not used for indentation Link
vue/no-shared-component-data When using the data property on a component (i.e. anywhere except on new Vue), the value must be a function that returns an object. Link
vue/no-side-effects-in-computed-properties It is considered a very bad practice to introduce side effects inside computed properties. It makes the code unpredictable and hard to understand. Link
vue/no-textarea-mustache Disallows the use of mustaches within textareas Link
vue/no-unused-vars Disallow unused variable definitions of v-for directives or scope attributes Link
vue/order-in-components Enforce order in Vue component data Link
vue/require-default-prop This rule requires default value to be set for each props that are not marked as required. Link
vue/require-prop-types In committed code, prop definitions should always be as detailed as possible, specifying at least type(s). Link
vue/return-in-computed-property Enforce that a return statement is present in computed property Link
vue/this-in-template Disallow using "this" in template form Link
vue/v-bind-style Enforce the shorthand v-bind: syntax (😃 Link
vue/v-on-style Enforce the shorthand v-on: syntax (@) Link

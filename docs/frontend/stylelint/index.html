# Stylelint

We make use of an automated approach of enforcing CSS code standards. These standards are constantly evolving to ensure we are making use of the best methods, and in order to catch more edge cases ensuring our code is consistent across the board.

# Rules

The following rules are currently configured.

Rule Description Link
block-no-empty Disallow empty blocks. Link
color-hex-case Sets the case of hex values to lowercase. Link
color-hex-length Forces the length of hex codes used in styles to be long notation rather than short. Link
color-named Colors must never be named. Link
color-no-invalid-hex Disallow invalid hex colors. Link
declaration-block-no-duplicate-properties Disallow duplicate properties within declaration blocks. This rule ignores variables ($sass, @less, --custom-property). Link
declaration-block-no-shorthand-property-overrides Prevents shorthand properties overriding longhand ones. Link
declaration-no-important Disallow !important within declarations. Link
indentation Always indent at-rules, rules, comments, declarations, inside parentheses and multi-line values by 4 spaces. Link
length-zero-no-unit Zero lengths should not have units. Link
max-empty-lines Limit the number of adjacent empty lines. Link
media-feature-colon-space-after Require a single space after the colon in media features. Link
no-extra-semicolons Disallow extra semicolons. Link
number-leading-zero Requires a leading zero on fractional values that are less than one. Link
property-case Specify lowercase for properties. Link
property-no-unknown Prevents unknown properties being used. Link
shorthand-property-no-redundant-values Disallow redundant values in shorthand properties. Link
unit-no-unknown Prevents unknown units being used. Link
